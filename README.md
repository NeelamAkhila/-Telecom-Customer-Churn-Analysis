# 📊 Telecom Customer Churn Analysis

## 📌 Project Overview

This project analyzes customer churn in the telecom industry to identify the factors influencing customer attrition and provide actionable business insights. The project follows a complete Data Analytics workflow using Python, MySQL, SQL, and Power BI.

---

## 🛠️ Tools & Technologies

- Python (Pandas, NumPy)
- MySQL
- SQL
- Power BI
- Git & GitHub

---

## 📂 Project Workflow

### 1. Data Collection
- Imported IBM Telco Customer Churn Dataset.

### 2. Data Cleaning (Python)
- Removed missing values.
- Converted data types.
- Handled blank values in TotalCharges.
- Performed Exploratory Data Analysis (EDA).

### 3. SQL Analysis
- Imported cleaned data into MySQL.
- Wrote **30 SQL queries** to analyze customer behavior, churn patterns, revenue, contracts, payment methods, and tenure.

### 4. Power BI Dashboard
Created an interactive dashboard with:
- Total Customers
- Churn Rate
- Monthly Revenue
- Average Monthly Charges
- Average Tenure
- Churn Distribution
- Churn by Contract Type
- Churn by Internet Service
- Monthly Charges vs Churn
- Churn by Tenure Group

---

## 📊 Key Business Insights

- Overall churn rate is **26.54%**.
- Month-to-Month contract customers have the highest churn.
- Fiber Optic customers churn more than DSL customers.
- Customers with lower tenure are more likely to churn.
- Customers with higher monthly charges have a higher probability of churn.
- Long-term contract customers are more likely to stay with the company.

- Encourage customers to move from Month-to-Month to long-term contracts.

- Review pricing strategies for Fiber Optic plans.
